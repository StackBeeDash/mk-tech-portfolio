# Skill: decompose-issues

## Purpose
大粒の Issue を実装可能な粒度に分解し、マイルストーンに振り分ける。

## Trigger
ユーザーが `/decompose-issues` と入力したとき、または Issue の分解を依頼したとき。

## Workflow

### Step 1: Issue の分析
- GitHub Issue の本文を読み取る
- 要件を機能単位に分解する
- 依存関係を特定する

### Step 2: サブタスク生成
各サブタスクに以下を含める:
- タイトル（動詞で始める: "実装する", "追加する", "設定する"）
- 受け入れ基準（チェックリスト形式）
- 推定工数（S/M/L）
- 依存する他タスク

### Step 3: マイルストーン振り分け
- 依存関係の順序を尊重する
- 1マイルストーンあたり 3-7 Issue を目安にする
- 各マイルストーンが独立してリリース可能か確認する

## Output Format
```markdown
## 分解結果

### マイルストーン: v1.0 - 基盤構築
- [ ] #XX タイトル (S) — 依存なし
- [ ] #XX タイトル (M) — #XX に依存

### マイルストーン: v1.1 - 機能実装
- [ ] #XX タイトル (L) — #XX, #XX に依存
```

## Example Prompt
```
以下の Issue を実装可能な粒度に分解してください:
- Issue #10: ユーザー認証システムの実装
目標: 3つのマイルストーンに分けて段階的にリリースする
```

## Notes
- 1つの Issue が 2日以上かかりそうなら、さらに分解を検討する
- フロントエンド / バックエンド / インフラの作業は別 Issue にする
- テストは実装 Issue に含める（別 Issue にしない）
