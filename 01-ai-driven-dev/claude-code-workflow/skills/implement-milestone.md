# Skill: implement-milestone

## Purpose
マイルストーンに紐づく全 Issue を一括で実装する。

## Trigger
ユーザーが `/implement-milestone` と入力したとき。

## Workflow

### Step 1: マイルストーンの確認
- GitHub API でマイルストーンの Issue 一覧を取得する
- 各 Issue の依存関係と優先度を確認する
- 実装順序を決定する

### Step 2: 実装計画の提示
```markdown
## 実装計画: v1.1 - AI & Claude Code

実装順序:
1. #13 Claude Code ワークフロー設計 (依存なし)
2. #14 マルチエージェントシステム (依存なし)
3. #15 MCP サーバー統合 (依存なし)
4. #16 業務自動化サンプル (依存なし)
5. #17 導入ガイド (#16 の後が望ましい)

推定所要時間: 約 2-3 時間
```

### Step 3: 逐次実装
各 Issue について:
1. Issue 本文の「構成案」に従ってディレクトリとファイルを作成
2. コードを実装（動作可能な品質）
3. README に Why を記載
4. 全 Issue 完了後に 1 コミットにまとめる

### Step 4: 完了報告
- 作成したファイル一覧
- 各 Issue の実装サマリ
- コミットハッシュ

## Configuration
```yaml
# 実装時のルール
rules:
  - 機密情報を含めない（APIキー、社名、顧客データ）
  - デモ用のダミーデータを使用する
  - 各 README に「なぜこの設計にしたか」を記載する
  - コードは実際に動作可能な品質にする
```

## Notes
- 大規模なマイルストーンでは、サブエージェントに並列実行を委譲できる
- 実装中に設計上の疑問が生じたら、Issue にコメントして確認を取る
- テストの自動実行が設定されている場合は、実装後にテストを走らせる
