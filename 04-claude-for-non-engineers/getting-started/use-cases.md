# Claude Code ユースケース集

部門・職種別の Claude Code 活用例。非エンジニアでも即日始められるユースケースを中心に紹介する。

## 営業・マーケティング部門

### 1. 顧客データの分析
```
> この CSV ファイルの顧客データを分析して、以下を教えて:
> - 月別の新規顧客数の推移
> - 顧客単価の分布
> - 解約率が高い顧客セグメントの特徴
```

**効果**: Excel での手作業 2-3 時間 → Claude Code で 5 分

### 2. 提案書のドラフト作成
```
> 以下の情報をもとに、提案書のドラフトを作成して:
> - 顧客: 製造業、従業員 500 名
> - 課題: 在庫管理の非効率
> - 提案: クラウド在庫管理システム
> - 予算感: 月額 50 万円以内
```

### 3. 競合分析レポート
```
> competitors.csv のデータをもとに、競合分析レポートを Markdown で作成して。
> 価格、機能、ターゲット顧客の3軸で比較表を含めて。
```

## 人事・総務部門

### 4. 社内規定の Q&A ボット作成
```
> この就業規則 PDF の内容をもとに、よくある質問と回答を 20 個作成して。
> カテゴリ: 勤怠、休暇、福利厚生、評価制度
```

### 5. 面接評価シートの作成
```
> エンジニア採用の面接評価シートを作成して。
> 評価項目: 技術力、コミュニケーション、問題解決力、カルチャーフィット
> 各項目に 5 段階の評価基準と質問例を含めて。
```

### 6. 議事録の自動生成
```
> この文字起こしテキストから議事録を作成して。
> フォーマット: 日時、参加者、議題、決定事項、アクションアイテム（担当者と期限付き）
```

詳細なサンプルコードは [meeting-minutes](../business-automation/meeting-minutes/) を参照。

## 経営企画・経理部門

### 7. 月次レポートの自動生成
```
> sales_2026_q1.csv と expenses_2026_q1.csv を読み込んで、
> Q1 の月次レポートを作成して。
> 含める内容: 売上推移、経費内訳、営業利益率、前年同期比
```

### 8. 予算策定の補助
```
> 過去3年分の実績データ (budget_history.csv) を分析して、
> 来期の部門別予算案を作成して。
> 前提: 売上成長率 15%、人件費上昇率 5%
```

### 9. KPI ダッシュボードの作成
```
> kpi_data.csv のデータを使って、経営ダッシュボード用のグラフを作成して。
> Python + matplotlib で、以下の 4 つのグラフ:
> 1. 月次売上推移（棒グラフ）
> 2. 部門別利益率（円グラフ）
> 3. 顧客獲得コスト推移（折れ線グラフ）
> 4. NPS スコアの推移（折れ線グラフ）
```

## カスタマーサポート部門

### 10. FAQ の自動生成・更新
```
> support_tickets_march.csv の問い合わせデータを分析して:
> 1. よくある問い合わせ TOP 10 を抽出
> 2. 各問い合わせに対する回答テンプレートを作成
> 3. 既存の FAQ (faq.md) を更新
```

### 11. 問い合わせの自動分類
```
> 新しい問い合わせメール (inquiry.txt) を読んで:
> - カテゴリ（技術的質問 / 料金 / 機能要望 / 不具合報告）を判定
> - 緊急度（高/中/低）を判定
> - 推奨対応（テンプレート回答 / エスカレーション / 開発チームへ転送）を提案
```

詳細なサンプルコードは [slack-monitor](../business-automation/slack-monitor/) を参照。

## エンジニアリングマネージャー

### 12. スプリントレトロスペクティブの分析
```
> retro_notes.md のレトロスペクティブのメモを分析して:
> - チームの課題を重要度順にランキング
> - 前回のレトロからの改善状況を評価
> - 次のスプリントで取り組むべきアクションを 3 つ提案
```

### 13. 技術負債の可視化
```
> このリポジトリのコードを分析して、技術負債のレポートを作成して:
> - 複雑度が高いファイル TOP 10
> - テストカバレッジが低い領域
> - 非推奨ライブラリの使用箇所
> - 推奨される対応優先順位
```

### 14. チームの開発パフォーマンス分析
```
> GitHub の PR データをもとに、チームのパフォーマンスレポートを作成して:
> - PR のリードタイム（作成→マージ）の推移
> - レビュー待ち時間の分布
> - 個人別の貢献度（PRs, Reviews, Comments）
> - DORA メトリクスの算出
```

## Tips: より良い結果を得るために

### 1. コンテキストを十分に提供する
```
# Bad
> 売上を分析して

# Good
> sales_2026.csv の月別売上データを分析して、
> 前年同期比の成長率と、季節変動のパターンを教えて。
> グラフは matplotlib で作成して、日本語フォント対応してね。
```

### 2. 出力フォーマットを指定する
```
> 結果は以下のフォーマットで出力して:
> - Markdown テーブル形式
> - 金額は ¥XX,XXX 形式
> - パーセンテージは小数点1位まで
```

### 3. 段階的に依頼する
```
# Step 1
> まず CSV ファイルの構造を確認して、カラムの意味を教えて

# Step 2
> データのクレンジング（欠損値、異常値の処理）を行って

# Step 3
> クレンジング後のデータで分析して、レポートを生成して
```

### 4. CLAUDE.md でプロジェクトのルールを定義する
繰り返し使う設定（出力形式、使用ライブラリ、命名規則等）は CLAUDE.md に記述しておくと、毎回指示する必要がなくなる。
